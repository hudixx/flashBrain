# FlashBrain 系统交互说明文档 (IXD)

**版本：** v1.0  
**文档类型：** 交互设计规范 (Interaction Specification)  
**适用对象：** 前端开发、测试工程师、UI 设计师

---

## 1. 全局操作规范 (Global Operations)

系统遵循“跨端一致，操作差异”的原则，针对 PC 端的高效输入和移动端的便捷手势进行了差异化设计。

| 交互动作 | PC 端 (鼠标/键盘) | 移动端 (手势/触摸) | 逻辑反馈 |
| :--- | :--- | :--- | :--- |
| **置顶 (Pin)** | 点击卡片右上角“大头针”图标。 | 右滑卡片呼出菜单，点击“置顶”。 | 卡片移至顶部，背景变为淡蓝色，带置顶标识。 |
| **标记掌握 (Master)** | 点击卡片操作栏的“勾选”图标。 | **左滑卡片到底** (核心手势)。 | 卡片变灰且文字减淡，自动取消置顶，原地留存。 |
| **搬家归档 (Archive)** | 点击“转移”图标或拖拽至侧边栏目录。 | 右滑菜单，选择“转移”。 | 播放“飞向侧边栏”动画，卡片从主视图物理移除。 |
| **批量操作 (Batch)** | `Shift` / `Ctrl` + 点击卡片。 | 长按任意卡片进入多选模式。 | 底部浮现操作栏：支持批量置顶、变灰、转移。 |

## 2. 编辑器交互逻辑 (Editor Interaction)

### 2.1 OCR 处理流
- **触发机制**：图片上传成功后即刻触发后台 OCR 任务。
- **视觉反馈**：OCR 文本框进入“识别中...”加载状态。
- **自动清洗**：识别文本需经 AI 预处理（去除杂质、纠正排版），完成后自动填充进入原文区。

### 2.2 独立保存策略
- **原文区保存**：点击保存仅更新 `ocr_text` 字段，提示：“原文存档成功”。
- **笔记区保存**：点击保存更新 `note_content`。支持自动保存机制，右上角显示“已自动保存”。
- **互不干扰**：任何一侧的保存动作不得刷新另一侧的编辑状态。

### 2.3 跨科目浮窗预览 (Popover)
- **触发**：点击卡片底部的非当前科目标签（如在 `#数学` 视图点击内容下的 `#物理` 标签）。
- **展示**：弹出 **400x300px** 的悬浮窗，展示该知识点在目标科目下的排序位置及上下文。
- **消失**：点击浮窗外部区域或按 `Esc` 键。

## 3. 排序与显示算法 (Logic & Algorithms)

### 3.1 局部置顶排序 (Lexical Ordering)
系统使用浮点数 $sort\_order$ 字段。当用户将卡片 A 拖动到卡片 B 和卡片 C 之间时：
$$new\_order = rac{order\_B + order\_C}{2}$$
- **优点**：无需更新数据库中其他卡片的索引，仅需修改 A 的权重。

### 3.2 变灰显隐逻辑 (Visibility Toggle)
- **状态一 (开关开启)**：显示所有内容，已掌握（变灰）卡片按时间戳混排在流中。
- **状态二 (开关关闭)**：过滤掉所有 `is_mastered = true` 的卡片。

### 3.3 搜索穿透逻辑
- **激活搜索**：当用户在搜索框输入内容时，系统自动**无视显隐开关**，显示全库匹配项。
- **结果区分**：搜索结果中，归档项需特殊标注（例如：`来自：#数学/归档库`）。

## 4. 视觉反馈规范 (Visual Feedback)

- **反馈时长**：所有点击反馈（Ripple effect）应在 **200ms** 内响应。
- **空状态**：当科目下无内容时，展示“点击 + 开始记录第一个知识片段”的引导图。
- **同步状态**：顶部导航栏右侧显示“已同步”或“正在同步...”微标。

---

## 5. 跨平台布局标准

- **PC 端**：固定三栏布局。左侧侧边栏宽度 260px-300px（可收起）。
- **Pad 端**：支持分屏（Split View），列表与编辑器按 1:1 或 1:2 比例分布。
- **手机端**：默认展示卡片流，点击卡片全屏进入编辑器。
