# 详情页可调宽与全屏编辑设计

日期：2026-06-05

## 背景

FlashBrain 当前主界面采用三栏布局：左侧科目栏、中间片段列表、右侧详情编辑区。详情编辑区承载图片预览、OCR 原文、个人 Markdown 笔记预览与编辑等核心内容，但作为右侧边栏显示时横向空间偏小，影响阅读、校对和长文本编辑效率。

本设计目标是在不改变后端接口和数据结构的前提下，让用户在保持三栏上下文的同时，可以获得更大的详情页可视面积，并在需要深度编辑时进入真正全屏的详情编辑模式。

## 目标

- 默认三栏布局中，允许用户把详情编辑区调整得更宽。
- 提供“全屏编辑”模式，让详情页占满应用窗口。
- 保持现有知识片段的创建、选择、保存、上传 OCR、置顶、掌握和删除流程不变。
- 记住用户调整过的布局宽度，刷新页面后恢复。
- 实现范围聚焦在前端，不改后端、OCR 服务和数据库结构。

## 非目标

- 不重新设计科目树或片段列表的信息架构。
- 不引入新的路由页面或独立详情页 URL。
- 不改变 OCR、笔记保存、片段状态切换等 API。
- 不实现复杂的多面板布局管理器，只提供本次需要的列表宽度拖拽和详情全屏。

## 推荐方案

采用“A + B”组合：

1. **默认三栏可调宽**：保留左侧科目栏、中间片段列表、右侧详情编辑区，在片段列表和详情区之间增加拖拽分隔条。用户可以拖窄片段列表，让详情编辑区获得更多横向空间。
2. **详情真正全屏**：在 `DoubleEditor` 顶部增加“全屏编辑”按钮。点击后隐藏左侧科目栏和中间片段列表，让详情编辑区占满应用窗口；再次点击“退出全屏”恢复三栏。

## 布局行为

### 默认三栏状态

布局结构保持为：

```text
左侧科目栏 | 中间片段列表 | 可拖拽分隔条 | 右侧详情编辑区
```

行为要求：

- 左侧科目栏保持固定宽度，例如当前的 `280px`。
- 中间片段列表使用可调宽度，默认值建议为 `360px`。
- 右侧详情编辑区使用剩余空间。
- 拖拽分隔条位于片段列表和详情编辑区之间。
- 片段列表最小宽度建议为 `260px`，避免标题、按钮和列表内容完全不可用。
- 详情编辑区最小宽度建议为 `480px`，避免拖拽后编辑区再次变得不可读。
- 用户拖拽结束后，将片段列表宽度保存到 `localStorage`。
- 页面刷新后优先读取 `localStorage` 中保存的宽度；若没有保存值，则使用默认宽度。

### 全屏编辑状态

布局结构变为：

```text
详情编辑区
```

行为要求：

- 隐藏左侧科目栏。
- 隐藏中间片段列表。
- 隐藏或移除三栏之间的拖拽分隔条。
- `DoubleEditor` 占满应用主窗口。
- 详情页内部继续显示图片预览、OCR 原文、个人笔记、Markdown 预览和操作按钮。
- `DoubleEditor` 顶部按钮从“全屏编辑”切换为“退出全屏”。
- 点击“退出全屏”后恢复三栏状态，并继续使用进入全屏前保存的片段列表宽度。

## 组件设计

### `MainLayout.vue`

`MainLayout.vue` 负责页面级布局状态。

新增职责：

- 维护 `isEditorFullscreen` 状态。
- 维护片段列表宽度，例如 `snippetPanelWidth`。
- 初始化时从 `localStorage` 读取片段列表宽度。
- 处理分隔条拖拽开始、拖拽中和拖拽结束。
- 在拖拽结束后保存宽度到 `localStorage`。
- 根据 `isEditorFullscreen` 决定是否渲染或隐藏左侧科目栏和中间片段列表。
- 向 `DoubleEditor` 传递当前全屏状态。
- 监听 `DoubleEditor` 发出的全屏切换事件。

建议事件边界：

```vue
<DoubleEditor
  :snippet="currentSnippet"
  :fullscreen="isEditorFullscreen"
  @toggle-fullscreen="isEditorFullscreen = !isEditorFullscreen"
  @updated="fetchSnippets"
/>
```

### `DoubleEditor.vue`

`DoubleEditor.vue` 继续专注于详情编辑自身，不直接管理三栏布局。

新增职责：

- 接收 `fullscreen` prop。
- 在编辑器标题栏增加布局按钮。
- 按钮文案根据状态显示：
  - 非全屏：`全屏编辑`
  - 全屏：`退出全屏`
- 点击按钮时发出 `toggle-fullscreen` 事件。
- 当没有选中 `snippet` 时，不显示或禁用全屏按钮。

现有职责保持不变：

- 图片上传和 OCR 调用。
- OCR 原文保存。
- 个人笔记保存。
- Markdown 预览。
- 置顶、掌握、删除操作。

## 状态与数据流

```text
用户拖拽分隔条
  -> MainLayout 计算新的片段列表宽度
  -> 更新 snippetPanelWidth
  -> 拖拽结束后写入 localStorage
  -> CSS 宽度变化使详情区获得更多空间

用户点击全屏编辑
  -> DoubleEditor emit('toggle-fullscreen')
  -> MainLayout 切换 isEditorFullscreen
  -> 左侧科目栏和片段列表隐藏
  -> DoubleEditor 占满应用窗口

用户点击退出全屏
  -> DoubleEditor emit('toggle-fullscreen')
  -> MainLayout 切换 isEditorFullscreen
  -> 恢复三栏布局
  -> 使用 snippetPanelWidth 恢复列表宽度
```

## 样式策略

- 优先使用现有 scoped CSS 风格，不引入新的 UI 框架。
- 拖拽分隔条使用窄条样式，默认低调，hover 时高亮，提示可拖动。
- 全屏状态可通过 `main-layout fullscreen-editor` 或类似 class 控制。
- 全屏下 `DoubleEditor` 的高度应适配视口，内部内容可滚动。
- 保持 Element Plus 按钮和图标风格一致。

## 边界情况

- 未选中片段：显示现有空状态，不提供全屏编辑入口。
- 浏览器宽度较窄：拖拽计算需要遵守最小宽度约束，避免列表或详情区被压到不可用。
- `localStorage` 中保存了异常值：初始化时进行数值校验，超出范围时回退默认宽度。
- 全屏状态下刷新页面：第一版不要求持久化全屏状态，刷新后回到默认三栏；宽度偏好仍保留。
- 删除当前片段：删除成功并刷新列表后，若当前片段不存在，应回到空状态；若此时处于全屏，可退出全屏或保持空状态全屏，推荐退出全屏以避免用户迷失上下文。

## 验证标准

完成后应手动验证：

1. 默认进入页面时仍显示左侧科目栏、中间片段列表和右侧详情编辑区。
2. 拖动片段列表和详情区之间的分隔条后，详情区宽度发生变化。
3. 刷新页面后，拖拽后的列表宽度能够恢复。
4. 点击“全屏编辑”后，左侧科目栏和中间片段列表隐藏，详情编辑区占满应用窗口。
5. 点击“退出全屏”后，三栏布局恢复。
6. 全屏和非全屏状态下，保存 OCR、保存笔记、上传图片、置顶、掌握、删除功能仍可正常使用。
7. 没有选中片段时，不出现可误导用户的全屏编辑入口。

## 实施范围

预计只需要修改：

- `frontend/src/layout/MainLayout.vue`
- `frontend/src/components/DoubleEditor.vue`

不需要修改：

- 后端 Java 代码。
- OCR 服务代码。
- 数据库实体。
- REST API 路径或请求体。
