# OCR 服务端口管理与自动重启设计规格说明书

## 1. 目标
优化 `ocr-server/main.py` 的启动逻辑，固定使用 `8093` 端口，并实现智能的端口占用处理机制（识别并重启自身或提示外部占用冲突）。

## 2. 详细设计

### 2.1 依赖引入
- **库**: `psutil`
- **用途**: 获取系统网络连接状态（监听端口）和查询特定 PID 的进程详细信息（命令行参数）。

### 2.2 端口与进程管理逻辑流程
当脚本启动时，执行以下步骤：
1. **端口扫描**: 检查本地 `8093` 端口是否处于 `LISTEN` 状态。
2. **决策树**:
   - **未占用**: 正常跳转至第 3 步启动服务。
   - **已占用**:
     - 获取占用该端口的 PID。
     - 获取该 PID 进程的命令行参数 (`cmdline`)。
     - **身份匹配**: 检查 `cmdline` 是否包含关键字 `main.py`（针对本项目脚本）。
     - **匹配成功 (自己)**: 
       - 输出重启提示。
       - 强制终止 (`kill`) 对应 PID。
       - 循环等待端口释放（设置超时机制）。
       - 跳转至第 3 步启动服务。
     - **匹配失败 (其他)**:
       - 输出错误提示，列出冲突 PID。
       - 终止启动流程。
3. **启动服务**: 使用 `uvicorn.run(app, host="0.0.0.0", port=8093)` 启动。

### 2.3 异常处理
- 若 `psutil` 无法获取某些特权进程的信息，应捕获异常并视为“外部应用占用”。
- 端口释放等待应有超时时间（如 5 秒），防止陷入死循环。

## 3. 验收标准
- 首次运行：成功启动在 8093。
- 再次运行（无修改）：旧进程被杀掉，新进程成功启动。
- 端口被其他软件（如浏览器或其它 Python 脚本）占用：提示启动失败并显示 PID。
